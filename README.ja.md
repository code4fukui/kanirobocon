# kanirobocon

越前がにロボコン（カニロボコン）のクライアントサイド得点記録システムです。iPadおよびPC向けに設計されています。サーバーサイドのコードは、福井県子どもプログラミング協議会のアカウントで別途公開されています: [fkpc/kanirobocon-system](https://github.com/fkpc/kanirobocon-system)。

このシステムは、参加者の受付（チェックイン）から決勝トーナメントの可視化まで、大会運営に必要なすべてのインターフェースを提供します。

## コアコンポーネント

- **得点記録インターフェース (`score.html`)**: 審判が試合中に2チームの得点をリアルタイムで記録するための詳細なインターフェース。
- **トーナメント表 (`tournament.html`)**: 公開用の自動更新される決勝トーナメント表。HTML5 Canvasを使用してトーナメントの枠、チーム名、接続線を描画し、結果が入力されると勝者を赤色でハイライトします。
- **対戦表生成 (`match1.html`, `match2.html`)**: 受付済みの参加者および決勝進出者に基づいて、予選および決勝の対戦スケジュールを自動生成するスクリプト。
- **結果とランキング (`result1.html`)**: 予選の結果とランキングを表示します。このページは、運営者が決勝トーナメントに進出するチームを生成（選定）するためにも使用されます。
- **管理メニュー (`menu.html`)**: 運営者向けのメインメニュー（ハブ）。内部用、公開用、管理用のすべてのページへのリンクを提供します。
- **スポンサー背景 (`sponsor.html`)**: `sponsor.csv`からスポンサーのロゴをシャッフルしてループ表示するシンプルなページ。会場のスクリーン等での公開表示を目的としています。

## 運営ワークフロー

本システムは、`menu.html`からアクセスし、以下の順序で使用するように設計されています：

1. **エントリー受付 (`entry.html`)**: 登録されたチームを大会に出席済み（エントリー済み）として記録します。
2. **予選対戦表の生成 (`match1.html`)**: 予選の対戦スケジュールを自動生成します。
3. **予選の得点記録**: スケジュールから試合をクリックして`score.html`を開き、試合結果を記録します。
4. **決勝トーナメントの生成 (`result1.html`)**: すべての予選が終了した後、ランキングを確認し、決勝トーナメントのリストを生成します。
5. **決勝の実施 (`match2.html`)**: 決勝トーナメント用の新しい対戦表が生成されます。リンクされた`score.html`を使用して、これらの試合の得点を記録します。
6. **トーナメント表の表示 (`tournament.html`)**: 決勝トーナメントの進行状況が観客向けに自動的に可視化されます。

## 技術的な留意点

- **バックエンドへの依存**: すべてのデータ操作は、`minidb`サーバーサイドコンポーネント（`kanirobocon-system`など）に依存しています。シンプルなAPIを介してCSVファイルとしてデータを読み書きします。エンドポイントは`minidb.js`で設定されています。
- **データ通信**: バックエンドとの通信は、クロスドメインリクエストを可能にするためにJSONPを使用して処理されます。APIキーはログイン（`index.html`）後に`localStorage`に保存されます。
- **互換性**: イベントで使用される古いiPadとの互換性を確保するため、モダンなJavaScript構文の使用には注意が必要です。
